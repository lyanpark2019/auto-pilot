# Design — Evals Harness cut-2.1 (Runnability & bounds)

> Sub-cut 1 of cut-2 (machinery-first). Parent spec: `2026-06-05-evals-harness-design.md`.
> Cut-1 landed (PR #20). This sub-cut stays **advisory** — no blocking gate.

## Purpose

cut-1 proved the clone→init→loop→oracle plumbing, but a real run **cannot currently
succeed** in a normal environment. The manual smoke (2026-06-06) returned `0/1` in 49s
with ~$0 spent. Root cause (verified, not guessed): the headless loop's fork-bomb guard
`check_caps` (`_budget.py:118-121`) calls `count_claude_pids()` = `pgrep -x claude`, which
counts **every** system `claude` process. With `default_max_concurrent_claude = 4`
(`_config.py:21`) and ≥4 unrelated `claude` sessions alive, the loop returns `cost-cap`
on iteration 1 **before any agent spawns** — no deliverable, oracle `fail`.

cut-2.1 makes a real eval run **possible** (cap-fix), **bounded** (cost/disk ceilings),
**honest** (real cost attribution), and **robust** (the cut-1 residual guards). Sequential
only — parallelism + `max_parallel_clones` is a later sub-cut.

## Locked decisions (do not relitigate)

1. **Sequential.** One case-loop at a time. No parallelism in 2.1.
2. **Advisory.** No blocking gate; `cli.main` still always exits 0.
3. **No core-file edits.** The cap-fix lives in the eval runner; `headless-loop.py` /
   `_budget.py` / `orchestrator.py` are untouched.
4. **Real validation is deferred.** The unit suite spawns no agent. A single cap-bypassed
   end-to-end run in a clean window confirms the fix; it is not a suite test.

## Architecture

### Cap-fix (the headline)
`runner.run_case` passes `--max-concurrent-claude _UNCAPPED_CONCURRENCY` (named module
constant, e.g. `10_000`, with a WHY comment) to `headless-loop.py`. Rationale: the
fork-bomb guard counts *all* system `claude` procs and is meant for a single interactive
PM loop; an eval **intentionally** spawns agents and runs one case-loop at a time, so it is
already bounded. `check_caps` uses `pids >= max_concurrent_claude`, so a large cap never
trips. (`--max-concurrent-claude` is a real flag — `headless-loop.py:292`.) No core edit
(the alternative — teaching `check_caps` a `0 = unlimited` sentinel — is cleaner but touches
a shipped core file; rejected for self-containment.)

### Real cost attribution
`_read_run_result` reads `state.get("cost_usd", 0.0)` from the clone's `state.json` (the
loop accumulates it per-iter at `headless-loop.py:224`; `State.cost_usd` exists —
`_state.py:67`). Replaces the `cost_usd=0.0` stub.

### run_case return contract change (verified-necessary)
`run_case` currently returns `OracleResult` and discards the `RunResult` (which carries
`cost_usd`/`iters`/`status`). To enforce a **total**-cost ceiling the CLI must see per-case
cost, so:

```python
@dataclass(frozen=True)
class CaseAttempt:      # in scripts/evals/_types.py
    oracle: OracleResult
    run: RunResult      # cost_usd, status, iters, ...
```

`run_case(...) -> CaseAttempt`. This ripples to `runner.py`, `tests/test_evals_runner.py`,
`cli.py`, and `aggregate.summarize` (now takes `list[CaseAttempt]`, can report cost + a
status distribution — useful for cut-2.5). The error path returns
`CaseAttempt(OracleResult("error", …), <best-effort RunResult>)`.

### Ceilings (fail-fast)
- **per-case cost** — already enforced (`--max-cost-usd 5.0`).
- **total corpus cost** — `cli.main` accumulates `attempt.run.cost_usd` **after each
  attempt** and checks the running total **before launching the next attempt/case**;
  stops launching further work when it exceeds `--max-total-cost-usd` (default **$50**, a
  named constant matching `_config.default_max_cost_usd`). Still exits 0 (advisory); reports
  the early stop in the results JSON.
- **disk** — `runner.run_case` checks `shutil.disk_usage(<tmp parent>).free` before
  `git clone`; if below `--min-free-disk-gb` (default **2** GB) it returns
  `CaseAttempt(OracleResult("error", "insufficient disk"), <zero-cost RunResult>)` without
  cloning. The error attempt carries a placeholder zero-cost `RunResult` (status `"error"`,
  dummy paths) so `summarize` stays uniform.

### Residual guards (cut-1 deferred #1/#2)
- `cli.main`: wrap the baseline read in try/except → malformed/missing baseline ⇒ advisory
  "no baseline" path, never a traceback.
- `stats.diff_upper`/`is_regression`: guard `n==0` (no `ZeroDivisionError`); `attempts==0`
  ⇒ not-armed / safe verdict.
- `aggregate.select_cases`: malformed `meta.json` ⇒ `ValueError(path)`, not a raw
  `JSONDecodeError`.

## Components / files

| file | change |
|---|---|
| `scripts/evals/_types.py` | add `CaseAttempt` frozen dataclass |
| `scripts/evals/runner.py` | cap const → `--max-concurrent-claude`; real cost via `state.get`; disk check; return `CaseAttempt` |
| `scripts/evals/aggregate.py` | `summarize(list[CaseAttempt])` (cost + status dist); `select_cases` meta guard |
| `scripts/evals/cli.py` | baseline-parse guard; total-cost accumulation + fail-fast; thread `CaseAttempt` |
| `scripts/evals/stats.py` | `n==0` guard |
| `tests/test_evals_*.py` | update runner/aggregate/cli tests for `CaseAttempt`; add ceiling + guard tests |

No `headless-loop.py` / `_budget.py` / `orchestrator.py` / CI changes.

## Testing

All TDD-mockable — mock `subprocess.run`, `state.json`, `shutil.disk_usage`; **zero agent
runs**, runs in the existing per-PR pytest gate. New tests: cap flag is passed; real cost
read from a fixture state.json; total-cost fail-fast stops further cases; disk-below-floor ⇒
error attempt; the three residual guards. One manual cap-bypassed end-to-end run (clean
window) validates the fix outside the suite.

## Out of scope (later sub-cuts)

Parallelism + `max_parallel_clones` · generic Verify-cmd oracle parsing (2.2) ·
promote/history (2.3) · fingerprint (2.4) · blocking-gate arming + measured baseline (2.5).
No process-tree-scoped pid counting (sequential makes it unnecessary).

## Open risks

- The cap-fix is validated by unit tests asserting the flag is passed, **not** by a real
  agent run in the suite. The end-to-end proof needs a clean window (no other `claude`
  sessions) and ~$1–5 — carried as a manual post-merge step.
- `CaseAttempt` changes a cut-1 public contract; all cut-1 callers/tests are updated in the
  same change (no external consumers exist yet).
- Cost is best-effort (log-scrape with estimate fallback upstream — `_budget.py:48`); the
  total ceiling inherits that fuzz.
